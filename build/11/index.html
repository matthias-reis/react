What's next?

That was a very basic introduction to React, the view layer of a possible frontend application. But several more aspects must be considered for that.

To name but a few:

  1. How can the composition of Components change over time (e.g. opening a lightbox, adding a success-message on top)?
    This should actually be easy because render() is called everytime something changes and therefore can return wht you want it to return
  2. How should we load code on demand?
    As soon as the user navigates to a different page / xing application, it is most likely that styles and code must be requested from the server. I'm pretty sure that we don't want to deliver the whole frontend on every fresh request.
  3. What about navigation? What happens when a user moves e.g. from the start page to his profile?
    • Fresh components (js) and styles (css) must be fetched (with techniques from (2))
    • The central area must display a different set of components change (with techniques from (1))
      • fetched (with techniques from (2))
    • The url in the browser must change
    • The history of the browser must be updated (to get browser back to work properly)
