Prop Types

Control over components parameters

For reusability reasons, you must be aware of the data you receive from your parent

  • With a prop type check, you can ensure, that only the right data type is transmitted
  • Usage 
    MyComponent.propTypes = {
  ...
}
  • Options
    • myArray: React.PropTypes.array For optional Arrays
    • Others:
      • React.PropTypes.bool
      • React.PropTypes.func
      • React.PropTypes.number
      • React.PropTypes.object
      • React.PropTypes.string
      also exist
    • myChild: React.PropTypes.element for React Components
    • myClass: React.PropTypes.instanceOf(MyClass) for special classes
    • myCount: React.PropTypes.oneOf([5, 10, 15]) restricts to certain values
    • myMessages: React.PropTypes.arrayOf(React.PropTypes.string) all array values must be strings
    • You can even define the types of object properties 
      myData: React.PropTypes.shape({
  color: React.PropTypes.string,
  fontSize: React.PropTypes.number
})
    • by default all values are optional, but you can add a required flag like so maxLength: React.PropTypes.number.isRequired
    • finally, you can also create your own validation by passing a function 
      customProp: function(props, propName, componentName) {
  if (!/matchme/.test(props[propName])) {
    return new Error('Validation failed!');
  }
}
