Data binding in React

The disadvantage of one-way-data-binding as in React is, that you potentially have to rewire everything everytime.

So if you want to update your state when the user changes an input, you need to

  • call an on change handler <input onChange={this.handleChange}
  • define the handler function 
    handleChange: function(event) {
  this.setState({value: event.target.value});
}
  • Apply the state to the input <input value={value} onChange={this.handleChange} />
  • As learnd from the past slides: If the Input component is not the stateful component, you even need to define handlers in one of their parent components (like Form)
  • It is very likely that ther will be a central piece of code to handle such repeititve stuff. Actually React (with addons) already provides a linkState mixin which is a simple solution
