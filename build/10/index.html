Lifecycle Methods

Create and destroy components

A component runs through several useful methods during its lifecycle:

  • componentWillMount(): occurs right before the initial rendering
  • componentDidMount(): occurs right after the initial rendering - here all initial work (like the integration with other frameworks or ajax calls) should be placed in here)
    • before the initial rendering
  • componentWillReceiveProps(props): occurs as soon as the parent component delivers new props (children included) - you can safely alter the component's state here based on the new props
  • shouldComponentUpdate(props, state): if this function returns false, the render() method will not fire - use this to avoid too many performance hungry repaints
    • new props
  • componentWillUpdate(props, state): triggered right before the re-rendering (not before the initial rendering)
    • new props
  • componentDidUpdate(props, state): triggered right after the re-rendering (not after the initial rendering) - use this to interact with other libraries
    • new props
  • componentWillUnmount(): triggered right before the component is removed from the dom. Use this to unset intervals, timers and other cleanup tasks
