State

Get data into your markup

Components have two ways to manage dynamic data: this.state and this.props. Let's start up with state:

  • State is an internal store, only valid for the current component.
  • Use the class constructor() method to set an initial value
  • In case of events and other actions, thst state can change. Use the method this.setState() for this.
  • Hand over an object that contains the changed properties
  • State internally is an immutable object. Everytime you call setState, a new object is created by merging the changed properties
  • By calling setState a repaint is automatically triggered
  • State properties are accessible by this.state.propertyName
  • You can use that for renering state properties
  • But: Never modify state properties directly like this.state.property = 'foo'. This brings view and data out of sync.
